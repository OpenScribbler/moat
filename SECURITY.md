# Security Policy

## Reporting Vulnerabilities in the Specification

If you discover a security vulnerability in the MOAT specification itself (e.g., a flaw in the hash algorithm, signing input construction, or verification logic that could be exploited by an attacker), please report it responsibly.

**Email:** openscribbler.dev@pm.me

Please include:

- Which section of the specification is affected
- A description of the vulnerability and its potential impact
- Steps to reproduce or a proof of concept, if applicable

We will acknowledge receipt within 48 hours and provide a timeline for resolution.

## Scope

This policy covers the MOAT specification document. Implementation-specific vulnerabilities should be reported to the maintainers of the affected implementation.
